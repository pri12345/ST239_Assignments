# Question 2: Planting your decision tree

## Introduction
In this task, we will construct a decision tree by hand to predict whether a student athlete should be selected for the top BUCS Rugby Team, using their historical data on training, fitness, attendance, and prior club experience.

## Part 1: Gini Impurity at Root Node
The formula for the Gini impurity is:
$$ \text{Gini} = 1 - \sum (p_i)^2 $$
Where $p_i$ is the probability of an item belonging to class $i$. Using all 25 observations:
- Class 0 (No Top Team): $n_0 = 13$
- Class 1 (Top Team): $n_1 = 12$
$$ p_0 = \frac{13}{25} = 0.52 $$
$$ p_1 = \frac{12}{25} = 0.48 $$
$$ \text{Gini(Root)} = 1 - (0.52^2 + 0.48^2) = 1 - (0.2704 + 0.2304) = 1 - 0.5008 = 0.4992 $$

## Part 2: DecisionTreeClassifier Output (max_depth=2)
If we were to use `DecisionTreeClassifier(DATA, criterion='gini', splitter='best', max_depth=2)`, the specific default rules generated by Python are:
- The first split attempts to find the absolute best split using all features. It determines that splitting on `TrainingHours <= 7.50` yields the best Gini drop.
- For `TrainingHours <= 7.50`, the next split is `TrainingHours <= 6.50`. Since max depth is 2, it results in predicting class 0 in both leaves for these thresholds.
- For `TrainingHours > 7.50`, the next split is `FitnessPass <= 0.50`. If `FitnessPass <= 0.50` it predicts class 0, and if `> 0.50` it predicts class 1.
So, the output flowchart of this unconstrained (other than max depth) tree is slightly different from forcing our first split on exactly 8 hours.

## Part 3: First Split Forced on TrainingHours
The coach requires $\ge 4$ sessions of 2 hours, meaning $\ge 8$ hours of training per week. We force our first split:
- **Left Child Node $(< 8 \text{ hours})$**: Contains 13 samples (IDs where `TrainingHours` < 8). 
  - Class distribution: 12 in Class 0, 1 in Class 1. 
  - $\text{Gini}_{\text{left}} = 1 - ((12/13)^2 + (1/13)^2) \approx 0.1420$
- **Right Child Node $(\ge 8 \text{ hours})$**: Contains 12 samples (IDs where `TrainingHours` $\ge 8$).
  - Class distribution: 1 in Class 0, 11 in Class 1.
  - $\text{Gini}_{\text{right}} = 1 - ((1/12)^2 + (11/12)^2) \approx 0.1528$

We then compute the weighted Gini impurity for the new split:
$$ \text{Weighted Gini} = \left(\frac{13}{25} \times 0.1420\right) + \left(\frac{12}{25} \times 0.1528\right) \approx 0.1472 $$
**Impurity reduction from this split**:
$$ \text{Reduction} = 0.4992 - 0.1472 = 0.3520 $$

## Part 4: Second-Level Split Evaluation
Next, we evaluate all other variables (`FitnessPass`, `AttendanceGood`, `PriorClub`) for creating a second split on both the left child and right child nodes to see which provides the purest nodes.

### Left Child Node (TrainingHours < 8, $n = 13$)
- **Split on FitnessPass**: Weighted Gini = 0.1319
- **Split on AttendanceGood**: Weighted Gini = 0.1346
- **Split on PriorClub**: Weighted Gini = 0.1231

The best split for the left branch is **PriorClub** (offers lowest weighted Gini of 0.1231).

### Right Child Node (TrainingHours $\ge 8$, $n = 12$)
- **Split on FitnessPass**: Weighted Gini = 0.0833
- **Split on AttendanceGood**: Weighted Gini = 0.1333
- **Split on PriorClub**: Weighted Gini = 0.1458

The best split for the right branch is **FitnessPass** (offers lowest weighted Gini of 0.0833).

**Justification**: We choose `PriorClub` for the Left Child, and `FitnessPass` for the Right Child, since they bring the maximum reduction in Gini Impurity (i.e. they result in the lowest individual weighted Gini sums among the 3 options).

## Question 1: Effects of a Daily Average Transformation
Imagine that instead of `TrainingHours` per week, we were provided with a daily average (e.g. `TrainingHours / 7`). This mathematically is a strictly monotonic linear transformation (division by a constant > 0). Decision Trees split data by ordering the values and finding the optimal cut point that lowers the impurity index. Because scaling a feature linearly preserves the exact relative ordering of all data points, the decision tree algorithm would select the exact same data split points (just scaled down by 7) resulting in the exact same impurity reductions, tree structure, and predictions. The predictions would not change at all.

Let's route the previous students through our new deep programmatic tree:

  `TrainingHours > 7.50` -> `FitnessPass <= 0.50` -> `AttendanceGood > 0.50`. This leaf predicts **Class 1**.

  `TrainingHours <= 7.50` -> `TrainingHours <= 6.50`. This leaf predicts **Class 0**.

## Model Comparison
- **Shallow Tree**: Our hand-made structure had 4 leaf nodes and forced an intuitive split first. It lacked full predictive power but successfully grouped wide chunks of the data with a high bias. It incorrectly classified Student A as 0 because it completely ignored their good attendance history once their fitness pass hit 0.
- **Deep Tree**: Expanded to 6 leaf nodes and a depth of 3, allowing full separation of the training data. It correctly leverages the combination of lack-of-fitness but good-attendance to classify Student A into the top team (Class 1).

**Why a Coach Might Prefer a Shallow Tree:**
A shallow tree provides simple, highly intuitive "rules of thumb" that are easy to explain verbally and memorize on the field. The coach might value interpretability over perfect classification, as highly deep trees often overfit to a small dataset (like our 25-samples dataset), learning noisy combinations that don't generalize to next year's crop of students. A shallow tree acts as a reliable heuristic that prevents overfitting.
